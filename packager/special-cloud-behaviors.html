Special cloud behaviors

info

This page is about the TurboWarp Packager.

This option changes the behavior of specifically named cloud variables to unlock new compatibilities for your project. This is based on a similar feature in HTMLifier.

To make these, just create a cloud variable as you normally would but give them the specific name found below. For example, to use the â˜ url variable, create a variable named url and mark it as a cloud variable.

Enabling special cloud behaviors will override any other settings for these variables, so a variable like â˜ username will never be stored locally or synced with other users.

â˜ urlâ€‹

The value of â˜ url will be set to the page's current URL. Changing the value of â˜ url does nothing.

â˜ redirectâ€‹

When the value of â˜ redirect is set to a URL, the current tab will navigate to that URL.

â˜ open linkâ€‹

When the value of â˜ open link is set to a URL, the project will attempt to open a new tab with that URL open. Note that this isn't always reliable due to the popup blockers built in to most browsers.

â˜ usernameâ€‹

When the value of â˜ username is changed, the value of the username block in the sensing category will change.

â˜ evalâ€‹

warning

This option requires "Unsafe special cloud behaviors" to be enabled.

Unsafe cloud behaviors allows the packaged project to execute arbitrary code outside of the "sandbox" that projects are typically executed in. Depending on the environment you're packaging for, this grants projects full control over your computer, including the ability to install viruses.

If you do not trust the project you're packaging or don't make use of this feature, please turn off this option.

When the value of â˜ eval is changed, its value will be evaluated as JavaScript.

If the JavaScript is successfully evaluated, its output will be stored in â˜ eval output.

If there was an error evaluating the JavaScript, the error will be stored in â˜ eval error.

If the JavaScript returns a Promise, it will store the resolved value in â˜ eval output if the promise resolves, or the error in â˜ eval error if it rejects. Note that setting â˜ eval is always an instant process, so the output variables may not update immediately.
